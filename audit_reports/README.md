## 🔄 版本管理规则

- **`v3.1/` 下 3 个核心MD是活文档** — 随项目迭代持续更新，每个新版本对应一个新文件夹（如 `v3.2/`）
- **`v3.1/reports/` 下 2 个报告是快照** — 对应特定版本，不随项目更新而修改
- **新版本时**：复制 `v3.1/` 内容到 `v3.2/`，在 `v3.2/` 中更新核心文件，重新生成 `v3.2/reports/` 报告

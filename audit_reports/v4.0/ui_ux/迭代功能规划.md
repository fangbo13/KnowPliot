### P0-1：Stop Generation 按钮

| 属性 | 值 |
|------|-----|
| **依据来源** | [来源: V4.0/ui_bug_list.md UI-HIGH-001] |
| **预估工期** | 1-2天 |
| **优先级理由** | 所有主流AI助手标配；当前30s超时对用户不可控；无需新依赖 |

**实现方案**：
- `ChatPage.tsx` 输入栏区域添加条件渲染：
  ```
  streamPhase !== 'idle' 时：
    显示 StopOutlined 按钮 (onClick = handleStop)
    替代 SendOutlined 按钮 (disabled)
  ```
- `handleStop` 函数逻辑：
  1. 调用 `abortActiveStream()` [StreamLifecycleManager.ts:37]
  2. 调用 `flushImmediate()` 保留已接收内容 [TokenBatchRenderer.ts:53]
  3. `set({ streamPhase: 'idle', streamContent: current })` — 保留已接收文本
  4. `clearStreamOnComplete()` 清理引用 [StreamLifecycleManager.ts:58]
  5. `unlockSend()` 解锁发送 [chatStore.ts:259]
  6. 不设置 sendError（abort是主动行为，不是错误）

**验收标准**：
- 流式期间：Stop按钮可见且可点击，Send按钮不可见
- 点击Stop：流式立即终止，已接收内容保留，无错误提示
- 点击Stop后再发送：正常工作，无残留状态

### P1-1：代码块复制按钮

| 属性 | 值 |
|------|-----|
| **依据来源** | [来源: V4.0/ui_bug_list.md UI-MED-002] |
| **预估工期** | 1天 |
| **优先级理由** | 代码复制是高频操作；全消息复制不够精细 |

**实现方案**：
- ReactMarkdown `components` prop 自定义 `pre` 和 `code` 渲染：
  ```tsx
  components={{
    pre: ({ children }) => (
      <div className="code-block-wrapper">
        {children}
        <CopyCodeButton />
      </div>
    ),
    code: ({ className, children, ...props }) => {
      const language = className?.replace('language-', '') || '';
      return <code className={className} {...props}>{children}</code>;
      // language 标签显示在 code-block-wrapper 顶部
    },
  }}
  ```
- `CopyCodeButton`：
  - absolute positioned 在 code-block 右上角
  - `navigator.clipboard.writeText(codeContent)` 复制仅代码文本
  - 桌面端hover显示，移动端始终显示
  - 点击后显示 `CheckOutlined` 2秒反馈

**验收标准**：
- 代码块右上角出现Copy按钮（hover显示/移动端常显）
- 点击复制仅代码块内容（非全消息）
- "Copied"反馈显示后消失
- 不干扰现有全消息复制功能

## 4. 长文本/代码块/特殊字符渲染压力测试场景

以下为 V4.0 需验证的渲染边界场景，每个场景标注预期问题：

### 场景 1: 长文本渲染（4000字符，含多级标题+嵌套列表）
- **输入**：含 h1-h6 + 3层嵌套ul/ol + blockquote 的4000字符Markdown
- **预期**：渲染正常（ALLOWED_ELEMENTS包含所有这些标签）
- **风险点**：消息气泡 maxWidth:75% 可能截断长标题

### 场景 2: 代码块渲染（100行Python，含缩进+注释+特殊字符）
- **输入**：````python\ndef foo():\n    ...100行...\n```
- **预期**：全灰色monospace纯文本，**无语法颜色区分**
- **根因**：ReactMarkdown无rehype-highlight → code内容原样输出
- **对比**：安装highlight.js后应有关键字紫色/字符串绿色/注释灰色

### 场景 3: GFM表格渲染（5列×10行数据表）
- **输入**：`| Column1 | Column2 | ... |`
- **预期**：渲染为原始管道符+文本，**非HTML表格**
- **根因**：remark-gfm未安装 → 表格Markdown语法不会被解析为table元素
- **对比**：安装remark-gfm后应渲染为对齐的HTML表格

### 场景 4: GFM删除线渲染
- **输入**：`~~deleted text~~`
- **预期**：渲染为带~~的原始文本，**非删除线效果**
- **根因**：remark-gfm未安装 → 删除线语法不生效

### 场景 5: GFM任务列表渲染
- **输入**：`- [x] done\n- [ ] todo`
- **预期**：渲染为普通列表项（`[x] done`、`[ ] todo`），**非复选框**
- **根因**：remark-gfm未安装

### 场景 6: 特殊字符渲染（HTML实体+Unicode emoji+CJK混合）
- **输入**：`&amp; &lt; 🎉 中文English混排`
- **预期**：ReactMarkdown的allowedElements白名单会strip不安全的HTML标签
- **风险点**：unwrapDisallowed=true 会移除不在白名单的标签wrapper

### 场景 7: 流式长代码块（流式期间收到50行代码）
- **输入**：流式SSE传输50行Python代码
- **预期**：流式期间为纯文本span（whiteSpace:pre-wrap），结束后1次ReactMarkdown解析
- **验证点**：流式期间代码块无高亮（符合预期），结束后也无高亮（BN-002 Bug）

### 场景 8: 移动端375px视口+长代码块
- **输入**：iPhone SE宽度下查看含50行代码的AI回复
- **预期**：代码块应在Card内横向滚动（overflow-x:auto）
- **风险点**：Card maxWidth:75% + padding:12px 16px 可能使代码块实际可用宽度不足
- **计算**：375px × 75% = 281px → 减去左右padding 32px = 249px可用宽度
  → 249px对50行代码可能不够 → 需overflow-x:auto生效

1 产品定位

KnowPilot V6.0 是面向专业服务团队的知识库应用 Agent。它把分散在文档、网页、项目经验和制度流程中的知识统一入库，通过 RAG 检索增强生成，为用户提供可追溯、可治理、可复用的知识问答入口。

RAG
检索增强生成，答案附来源引用
V6.0
当前版本口径，已从早期入职问答升级
Agent
自然语言入口，承接组织知识查询
Governed
权限、日志、系统健康与合规追踪

它解决什么问题

  • 知识分散：制度、准则、方法论、项目经验分布在 PDF、内网、文件夹和个人经验中，查找成本高。
  • 答案不一致：同一个问题依赖口头经验传递，容易出现不同团队不同口径。
  • 经验难沉淀：项目判断、会议纪要、客户背景和底稿逻辑沉在个人脑中，人员流动后知识断层。
  • 合规难追踪：知识资产需要被管理、授权、更新和审计，不能只靠普通文件共享。

为什么诞生

KnowPilot 诞生于专业服务团队真实的知识协作痛点：新人入职不知道从哪里找流程，项目成员面对大量底稿不知道历史判断依据，经理和合伙人需要快速复盘客户背景、准则条款和项目经验。传统方式依赖“问同事、翻文件、找群聊记录”，不仅慢，而且答案容易随人变化。

因此，KnowPilot 的目标不是再做一个普通 Chatbot，而是把组织知识沉淀成一个可持续运营的知识库应用 Agent。它让团队把制度、准则、方法论、项目经验、培训材料和网页资料统一入库，再通过对话式入口快速获取带来源的答案。

Before
新人靠“考古”上手
流程、模板、IT 设置和报销规则分散在不同地方，新人反复打断同事，团队培训成本高。
Problem
经验跟着人走
关键项目判断和客户沟通语境沉在个人经验里，一旦人员流动，组织经验就被迫归零。
After
知识变成组织资产
所有知识通过 RAG 入库、检索、引用和审计，团队可以持续复用、更新和治理。

能不能推广

可以推广，而且 KnowPilot 的优势正在于可复制。它采用“同一套知识库 Agent 引擎 + 不同团队知识空间”的方式：HR 团队可以维护入职宝典，审计团队可以维护方法论和准则库，项目组可以维护客户背景、会议纪要和历史判断，咨询团队可以维护行业方法论和交付标准。

当一个团队验证价值后，其他团队不需要重新开发系统，只需要建立自己的知识库、配置权限和持续维护内容即可复用。它适合推广到知识密集、合规要求高、重复问答多、经验传承成本高的团队和行业，例如审计、金融、咨询、医疗、政务和大型企业内部知识管理。

2 首页与问答入口

用户进入 KnowPilot 后，首先看到统一的知识问答入口。首页保留熟悉的聊天式交互，同时通过常用问题卡片降低首次使用门槛。

统一提问框 Agent
用户可以直接输入自然语言问题，不需要知道知识库中文档的具体位置。
常用问题卡片
把高频问题前置，例如 IT 设置、报销流程、年假天数、培训课程、办公位置、导师安排。
会话侧边栏
按日期组织历史会话，支持搜索、继续追问和知识查询上下文沉淀。
全局工具区
顶部提供语言、主题和用户入口，适配中英双语与亮暗主题。

3 知识问答 Agent

KnowPilot 的核心体验是基于知识库的可追溯问答：用户发问后，系统检索相关知识片段，调用模型生成答案，并把引用来源返回给用户。

功能说明状态

来源引用回答中保留文档引用，用户可以判断依据来自哪里。已实现
会话管理按会话保存历史问答，便于复盘和继续追问。已实现
错误与离线处理识别认证、网络、服务端等异常，避免静默失败。已实现

4 知识库管理

知识库管理页面向管理员或授权用户，负责把组织知识变成可检索、可引用、可维护的知识资产。

文档生命周期

上传文档
→
解析与切块
→
生成 Embedding
→
写入 pgvector
→
问答召回引用

管理员可执行操作

  • Upload：上传知识文档，支持常见文本资料入库。
  • Refresh：刷新当前知识库状态，查看新增、处理完成或失败的文档。
  • Reindex：对文档重新索引，适用于内容更新或索引异常恢复。
  • Delete：删除不再使用或不应继续提供问答依据的文档。
  • Web 采集：通过爬虫链路扩充外部知识，配套 SSRF 防护、User-Agent 和合规审计。

5 管理后台与系统健康

KnowPilot 不只提供问答，还提供系统治理能力。管理员可以查看用户、角色、服务状态和关键运行信息。

用户管理 Admin
展示用户邮箱、用户名、角色、业务线和启用状态。
角色标签
支持 Admin、HR、Employee 等角色识别，为后续细粒度 RBAC 提供治理基础。
系统健康
展示 Backend、Celery、Database 等核心服务状态，便于排障。
审计追踪
关键操作保留日志，帮助定位谁在何时修改了知识资产或权限配置。

6 端到端流程

从知识进入系统，到用户得到答案，KnowPilot V6.0 的核心链路如下。

准备知识源

业务团队准备 HR 手册、审计方法论、准则条款、项目经验、培训材料、FAQ 或网页资料。

知识入库

管理员通过上传或 Web 采集把资料加入知识库，系统完成解析、切块和向量化。

用户提问

员工、项目组或管理者在聊天入口提出自然语言问题。

RAG 检索

系统从 pgvector 中召回相关片段，并结合上下文组织答案。

生成与引用

模型返回答案，同时保留文档引用，用户可以追溯依据。

审计与维护

管理员根据使用反馈更新文档、重建索引、调整权限并追踪操作记录。

7 典型应用场景

KnowPilot 的价值来自“同一套知识引擎，多业务场景复用”。只要替换知识库内容，就能切换到不同团队的专属知识空间。

场景知识库内容典型问题
入职培训助手HR 入职宝典、IT 设置、报销流程、培训课程电脑和邮箱怎么配置？报销发票怎么分类？
审计项目组 AI项目关键信息、会议纪要、客户背景、判断逻辑这个调整为什么这么做？去年拒函的原因是什么？
审计方法论查询审计方法论、底稿模板、优秀范例TOC / TOD / WT 分别怎么做？抽凭标准程序是什么？
准则问答助手CAS、HKFRS、IFRS 等准则条款IFRS 15 收入确认五步法如何适用？
风险识别历年风险案例、行业特征、重大错报风险库制造业客户典型风险领域有哪些？
合规检查独立性规则、违规案例、内部政策团队成员持有客户股票应该怎么办？

8 角色与权限

知识库应用 Agent 必须有清晰的访问边界。KnowPilot 支持按业务线、项目或部署空间划分知识库负责人，让各团队可以自由管理自己负责的知识库，同时保留认证、角色和审计能力。

角色可执行操作说明
Employee提问、查看自己可访问的知识问答结果普通使用者，只接触授权范围内的知识。
Knowledge Owner管理所属业务线、项目或部署空间的知识库每条线或每个项目的负责人可自行上传文档、维护内容、刷新状态、重建索引和管理本空间知识资产。
Content Admin上传文档、维护知识库、刷新或重建索引协助负责人维护内容质量、知识更新和资料入库。
System Admin用户管理、角色配置、系统健康查看、审计追踪负责平台级治理、运行状态和跨空间权限边界。

个人资料与偏好

  • Account Information：展示用户名、邮箱、业务线、办公地点和角色等级。
  • Language Preference：支持用户选择界面语言，配合中英双语使用场景。
  • Save Changes：保存个人偏好，确保用户下次进入系统时延续相同体验。

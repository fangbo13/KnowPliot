## P0：立即可做（1-3天）— 致命问题修复

### P0-1：聊天 AI 响应延迟验证与修复

- **优先级**：🔴 最高
- **问题**：BUG-002 — SSE 流式响应在测试中 15s+ 无可见反馈
- **修复方案**：
  1. 在真实浏览器（非 headless）中手动验证聊天流式是否正常
  2. 如果 API 正常但前端显示有问题，检查 SSE 解析逻辑（chatStore.ts 的 sendMessage 中 `fetch()` + `text()` 模式）
  3. 添加即时反馈：发送后 <1s 显示 "正在连接..."，而非等待 10s 才显示思考指示器
  4. 添加 fallback：如果 SSE 5s 内无 token，切换到 POST polling 模式
- **预期改善**：聊天响应体验从"可能卡住"变为"始终有反馈"
- **涉及文件**：`ChatPage.tsx`, `chatStore.ts`

### P0-2：思考指示器即时显示

- **优先级**：🔴 高
- **问题**：UX-003 — 发送后 10s 才显示 "thinking..."，用户会以为系统故障
- **修复方案**：
  1. 发送后立即（<500ms）显示 animated dots + "正在思考..."
  2. 3s 后切换文案："AI正在检索知识库..."
  3. 8s 后切换："正在生成回答..."
  4. 取消当前的 10s 延迟机制，改为即时显示 + 渐进式文案
- **预期改善**：消除"系统卡住"的误解，增加用户信任
- **涉及文件**：`ChatPage.tsx`, `chatStore.ts`

## P1：短期规划（1-2周）— 遗留修复与核心交互优化

### P1-1：Profile 页面内容扩展

- **问题**：UX-001 — Profile 只有 email 和 language，极简但不完整
- **修复方案**：
  1. 添加 username 显示（可编辑或 disabled，视需求定）
  2. 添加 service_line、office_location、role_level 信息展示
  3. 添加默认头像展示
  4. 添加 "上次登录时间" 信息
  5. 重新设计 Profile 页面布局：左侧头像区 + 右侧信息区（桌面），上方头像+下方信息（移动）
- **预期改善**：Profile 从"空壳页面"变为"有用的个人中心"
- **涉及文件**：`ProfilePage.tsx`

### P1-2：移动端汉堡菜单优化

- **问题**：UX-002 — 移动端侧边栏触发不够明显
- **修复方案**：
  1. 增大汉堡按钮触控区域至 44×44px
  2. 使用更明显的 MenuOutlined icon（三条横线）
  3. 将汉堡按钮固定在 header 左侧（符合主流 App 习惯）
  4. 首次移动端访问时自动展开 Drawer 2s 让用户发现侧边栏
- **预期改善**：移动端导航体验从"难以发现"变为"直觉操作"
- **涉及文件**：`AppLayout.tsx`

### P1-3：Demo 账号一键填入

- **问题**：UX-004 — Login 页 demo 提示需要手动复制粘贴
- **修复方案**：
  1. 在 Info Alert 中添加 "Use Demo Account" action link
  2. 点击自动填入 email + password 并聚焦
  3. 或直接改为可点击的 credential 文字（点击自动填入）
- **预期改善**：Demo 登录从 5 步操作变为 1 步
- **涉及文件**：`LoginPage.tsx`

### P1-4：侧边栏搜索改进

- **问题**：UX-005 — 搜索不够突出
- **修复方案**：
  1. 搜索框始终显示在侧边栏顶部
  2. 添加 SearchOutlined 图标
  3. 添加 i18n placeholder
- **预期改善**：历史会话查找从"需要滚动"变为"直接搜索"
- **涉及文件**：`AppLayout.tsx`

## P2：中期规划（2-4周）— 性能优化与组件重构

### P2-1：新手引导渐进化

- **问题**：UX-006 — Onboarding modal 信息密度高，一次性展示
- **修复方案**：
  1. 改为 step-by-step tooltip tour（类似 Ant Design Tour 组件）
  2. 添加 "Skip" 文字链接
  3. 第一次只介绍 Chat，历史和知识库在使用后引导
- **涉及文件**：`AppLayout.tsx`

### P2-2：聊天消息操作可见性优化

- **问题**：hover 显示的 copy/share/regenerate 操作在移动端不可发现
- **修复方案**：
  1. 桌面端：hover 时显示操作按钮（已有）
  2. 移动端：消息底部始终显示简洁操作行（copy icon + regenerate icon）
  3. 长按操作保留为补充方式
- **涉及文件**：`MessageBubble.tsx`

### P2-3：History 页面增强

- **问题**：HistoryPage 搜索和筛选体验需要优化
- **修复方案**：
  1. 添加全文本搜索（搜索消息内容而非仅标题）
  2. 优化时间筛选标签视觉
  3. 添加 "最近对话" 快捷入口
- **涉及文件**：`HistoryPage.tsx`

### P2-4：知识库上传 UX 优化

- **问题**：上传进度和状态反馈不够直观
- **修复方案**：
  1. 添加拖拽上传支持
  2. 上传时显示进度条（Progress component）
  3. 处理完成后显示 chunk 数量和成功提示
  4. 批量上传支持（多文件同时上传）
- **涉及文件**：`KnowledgeBasePage.tsx`

## P3：长期规划（架构与体系）— 1-3月

### P3-1：状态管理优化

- 当前 chatStore 使用 Zustand，但 sessions 和 messages 之间存在竞态条件（切换会话时消息可能交叉显示）
- 建议优化：添加 optimistic update lock、debounce session switch、message batching

### P3-2：可访问性全面提升

- 当前已有基础（skip-to-content、aria-live、reduced-motion），但未通过 WCAG 2.1 AA 级认证
- 建议添加：focus trap in modal、keyboard shortcuts for chat operations、high contrast mode、screen reader testing with NVDA/VoiceOver

### P3-3：组件重构与设计系统统一

- globals.css 超过 600 行，建议提取为 CSS Modules 或 styled-components
- AntD token 覆盖散落在多处，建议集中到 theme config 文件
- 添加 Storybook 组件文档和 visual regression testing

### P3-4：性能优化

- 代码分割已有（vendor/antd/markdown chunks），但可以进一步优化：
  - 添加 route-based lazy loading（React.lazy + Suspense）
  - 图片/文件 lazy loading
  - SSE 连接复用（避免每次发消息重新创建 fetch 连接）
  - MessageBubble markdown 渲染优化（memo + 虚拟滚动）

### P3-5：错误监控与日志

- 当前错误处理完善但没有集中化的错误监控
- 建议添加：Sentry 或类似 error tracking 服务
- 前端 performance monitoring（Web Vitals）
- 后端 API health check dashboard

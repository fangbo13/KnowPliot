## 1. 执行摘要

本次 V4.1 SYS 领域审计覆盖了两大维度：

1. **V4.0 修复回归测试**: 对 V4.0 修复的 7 个安全功能点（DEFECT-001/002/006/007/008/012/013）进行回测验证
2. **V4.1 新缺陷深度挖掘**: 对 CORS、DEBUG 配置、SSL、Celery、限流、并发、JWT 存储等维度进行新增缺陷发现

**关键发现**:

- V4.0 修复在 V4.1 中 **全部保留且有效**，无回归（6/8 PASS，2/8 PARTIAL PASS）
- V4.1 发现 **13 个新缺陷**（3 P1 + 3 P2 严重 + 7 P2 一般），其中 3 个 P1 阻断级缺陷来自 `docker.py` 配置文件
- 3 个 P1 阻断缺陷（CORS_ALLOW_ALL_ORIGINS、DEBUG=True、SSL_VERIFY=False）**共同构成攻击链**：任何 CORS → JWT 泄露 → 全 API 访问 → DEBUG → 完整系统信息泄露 → SSL → API Key 泄露

## 2. 稳定性评分

**V4.1 SYS 整体稳定性分**: **55/100** (🟠 Moderate Risk)

评分依据:

| 维度 | 分数 | 依据 |
|---|---|---|
| 架构健壮性 | 30/100 | 单进程 runserver、CONN_MAX_AGE=0、单 Celery Worker、无 timeout |

| 异步任务流 | 55/100 | Celery 配置基本正确，但无 timeout/分布式锁/多 worker |
| 数据库 | 50/100 | pgvector 正确使用，但连接无复用、无 health check |
| 资源管理 | 45/100 | Redis 无密码/maxmemory、Celery 无 timeout、文件上传限制不明确 |
| 前端状态 | 80/100 | Zustand 状态机 + React.memo + BroadcastChannel 机制完善 |

**提升路径**: 替换 runserver → gunicorn + CONN_MAX_AGE=60 + Celery timeout → 稳定性可达 75/100

## 3. 安全评分

**V4.1 SYS 整体安全分**: **35/100** (🔴 High Risk)

评分依据:

| 维度 | 分数 | 依据 |
|---|---|---|
| XSS 防护 | 90/100 | 三层纵深防御（ALLOWED_ELEMENTS + 协议校验 + unwrapDisallowed），V4.0 修复有效 |
| CSRF 防护 | 40/100 | JWT 认证天然防 CSRF，但 CORS_ALLOW_ALL_ORIGINS=True 否定了跨域防护 |
| 信息泄露 | 20/100 | DEBUG=True 泄露堆栈 + ALLOWED_HOSTS=["*"] + SECRET_KEY 短 + SSL_VERIFY=False |
| Rate Limiting | 55/100 | SendMessageRateThrottle 有效，但无 IP 限流、无登录限流 |
| 输入清洗 | 80/100 | Django ORM 参数化查询、前端白名单渲染、Serializer max_length=4000 |
| 认证/授权 | 70/100 | JWT + RBAC 双轨权限，但 localStorage 存储 + CORS 全开放 + JWT Key 短 |

**扣除因素**:
- CORS_ALLOW_ALL_ORIGINS=-20（任何域名可跨域请求）
- DEBUG=True=-15（完整堆栈泄露）
- SSL_VERIFY=False=-10（API Key MITM 截获）
- SECRET_KEY 26 bytes=-10（JWT 伪造风险）
- 无 IP 限流=-10（批量攻击绕过）

### Top 5 风险

| # | 风险 | 严重程度 × 发生概率 | 攻击链 |
|---|---|---|---|
| 1 | CORS_ALLOW_ALL_ORIGINS | P1 × 高 | XSS → localStorage JWT → 任意域名 API 请求 → 数据窃取 |
| 2 | DEBUG=True 堆栈泄露 | P1 × 高 | 任何 500 错误 → 文件路径/SQL/settings/API Key 全部泄露 |
| 3 | SSL_VERIFY=False | P1 × 中 | MITM → DashScope API Key 截获 → 伪造 RAG 回复 |
| 4 | SECRET_KEY 26 bytes | P2 × 中 | JWT 伪造 → admin 权限 → 全系统控制 |
| 5 | 无 IP 限流 | P2 × 高 | 批量注册 → 每账号 10 msg/min → ¥40/min DashScope 成本 |

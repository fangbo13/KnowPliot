## 二、爬虫知识入库实现（P0，核心目标）

### 2.1 功能需求

**场景**: Admin/HR 通过输入 URL 或关键词，系统自动抓取外部网页内容，经清洗、去重、向量化后入库到知识库，从而批量充实知识库覆盖度。

**关键能力要求**:
1. **单 URL 爬取**: Admin 输入一个 URL → 抓取 → 入库
2. **批量爬取**: Admin 输入一组 URL 或关键词 → 批量抓取 → 批量入库（重点，因为目标是大量信息）
3. **sitemap 发现**: 输入域名 → 自动发现 sitemap.xml → 批量爬取全站页面
4. **爬取状态追踪**: 爬取进度、成功/失败/跳过统计
5. **增量更新**: 已爬取的 URL 可重新爬取，更新内容版本

### 2.4 批量爬取架构（重点设计）

```mermaid
graph TD
    A[Admin 输入: URL列表 / 关键词 / 域名] --> B{类型判断}
    
    B -->|URL列表| C[批量创建 Celery Task: crawl_url.delay per URL]
    B -->|关键词| D[搜索引擎 API 搜索 → 获取 URL 列表 → C]
    B -->|域名| E[发现 sitemap.xml → 解析 URL 列表 → C]
    
    C --> F[Celery Worker 并发执行: max 4 concurrent]
    F --> G[URL 校验 + robots.txt + httpx Fetch]
    G --> H{爬取结果}
    
    H -->|成功| I[trafilatura 提取 + bleach 清洗 + SimHash 去重]
    H -->|失败| J[记录失败原因 + 可手动重试]
    H -->|跳过(重复)| K[记录跳过原因]
    
    I --> L[EmbeddingService 向量化 → pgvector 入库]
    L --> M[更新 CrawledDocument.crawl_status = 'active']
    M --> N[Admin Dashboard 实时刷新进度]
    
    style C fill:#4f4,stroke:#090
    style F fill:#4f4,stroke:#090
```

### 2.5 批量爬取性能考虑

| 参数 | 值 | 说明 |
|---|---|---|
| Celery 并发 | 4 workers | 不超载目标站点 |
| 单 URL 超时 | connect=30s, read=60s | 避免卡死 |
| 重定向上限 | 3 次 | SSRF 防护 |
| 内容大小上限 | 500KB/URL | 拒绝超大页面 |
| 批量上限 | 100 URLs/批次 | 单次提交最多 100 URL |
| 频率限制 | 10请求/域名/分钟 | 遵守 Crawl-delay |
| SimHash 阈值 | 3-bit difference | 近重复检测 |
| User-Agent | `EY-Onboarding-AI-Crawler/1.0 (+https://ey.com/bot)` | 可识别性 |

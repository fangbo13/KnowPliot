## 一、执行摘要

### 1.1 审计范围与方法

本次审计覆盖 EY Onboarding AI 系统的 **KB/Admin 领域**，在独立 Docker 环境（Frontend:3020, Backend:8020, DB:5434, Redis:6381）中执行。审计方法包括：

1. **API 级回归测试**：8 个越权 Payload + 4 个 HR KB 测试 + JWT claims 解码 + Token blacklist 验证 + 文件类型校验
2. **代码级深度审查**：permissions.py, users/models.py, retriever.py, knowledge/views.py, serializers.py
3. **爬虫安全规格分析**：全量代码搜索确认功能未实现，产出预实现安全规格书

### 1.2 关键发现

**⚠️ 最重要的发现**：V4.1 描述的"网络爬取知识入库"功能**在代码中完全不存在**。无爬虫端点、无 trafilatura/SimHash 库、无前端爬虫 UI。爬虫安全评分 0/100（功能未实现）。

**其他关键发现**：
- V4.0 RBAC 修复全部有效（8/8 越权测试通过）
- JWT claims 泄露 `is_hr_admin`（P0-2 残留）
- Token blacklist 失效（logout 后 token 仍可用）
- 无文件内容/magic number 校验
- retriever.py SQL 列名注入风险

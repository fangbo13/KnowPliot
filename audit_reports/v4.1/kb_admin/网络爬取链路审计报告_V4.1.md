## 一、爬虫架构概述

### 1.1 当前 httpx 使用现状

系统中 httpx.Client 的使用**仅限于 Embedding API 和 LLM Guardrails**：

| 用途 | 文件 | 连接目标 | 说明 |
|---|---|---|---|
| Embedding API | `backend/apps/rag/embedding.py` | DashScope API (dashscope.aliyuncs.com) | V3.7 全局共享连接池 |
| LLM Guardrails | `backend/apps/rag/guardrails.py` | DashScope API | LiteLLM 服务调用 |
| Playwright | `backend/run_tests.py` | 仅测试脚本 | 非业务代码 |

**关键结论**：httpx.Client 当前仅连接外部 LLM API 服务，**不用于网页抓取**。爬虫功能实现时需创建独立的 httpx.AsyncClient 或复用现有共享连接池，但需区分用途（外部 API vs 外部网页抓取）。

### 1.2 Celery 异步任务现状

系统已有 Celery Worker 运行（docker-compose 中 celery-worker 服务），当前仅处理文档入库任务 `ingest_document.delay()`。爬虫任务可作为新的 Celery Task 添加，与现有入库流程并行。

|---|---|---|
| 爬取 API 端点 | ❌ 无 | URL routing 中无 crawl/scrape 路径 |

| SimHash 库 | ❌ 无 | 无 simhash 相关代码 |

### SSRF 攻击场景（预规划级）

| Payload | 预期行为 | 无防护后果 |
|---|---|---|
| `http://127.0.0.1:8020/admin/` | 400（内网 IP 拦截） | 内网 Django Admin 页面被爬取 |
| `http://169.254.169.254/latest/meta-data/` | 400（云元数据拦截） | AWS/Azure 云凭据泄露 |
| `file:///etc/passwd` | 400（非法协议） | 服务器文件系统被读取 |
| `gopher://internal-redis:6379/` | 400（非法协议） | Redis 未授权命令执行 |
| `http://public-site.com/redirect-internal` | 400（重定向到内网） | 通过外网重定向绕过直接 IP 检查 |

## 七、安全规格要求表（Prompt 2B 开发约束）

### 7.1 P0 级要求（必须实现后才能上线）

| # | 要求 | 技术方案 | 验证方法 |
|---|---|---|---|
| SSRF-P0-1 | URL 协议白名单：仅 http/https | `validators.URLValidator(schemes=['http', 'https'])` | `file:///etc/passwd` → 400 |
| SSRF-P0-2 | 内网 IP 黑名单 | `ipaddress.ip_address()` 检查 + 黑名单集合 | `http://127.0.0.1` → 400 |
| SSRF-P0-3 | DNS Rebinding 防护 | 解析 hostname → 检查 IP → 发起请求 → 再次检查连接 IP | 重定向攻击 → 400 |
| SSRF-P0-4 | 重定向限制 ≤3 | `httpx.Client(follow_redirects=True, max_redirects=3)` | 4 次重定向 → 400 |
| CONTENT-P0-1 | HTML 清洗 | `bleach.clean(content, tags=ALLOWED_TAGS, attributes=ALLOWED_ATTRS)` | `<script>alert(1)</script>` → 清洗后无 script |
| CONTENT-P0-2 | 内容大小限制 ≤500KB | `if len(extracted_text) > 500_000: raise ValidationError` | 600KB 内容 → 400 |
| CONTENT-P0-3 | 非文本内容拒绝 | 检查 HTTP response Content-Type header | 图片/视频 URL → 400 |
| API-P0-1 | 爬虫端点权限 | `HasPermission("document.create")` | Employee POST → 403 |
| API-P0-2 | 新 AuditLog action | `ACTION_CHOICES` 新增 `"document_crawl"` | 爬取操作可审计 |
| MODEL-P0-1 | CrawledDocument 模型 | source_url, crawl_status, content_hash, document FK | 数据完整性 |

### 7.2 P1 级要求（上线后一周内实现）

| # | 要求 | 技术方案 | 验证方法 |
|---|---|---|---|
| SSRF-P1-1 | URL 长度限制 ≤2048 | `URLField(max_length=2048)` | 超 2048 字符 → 400 |
| COMPLIANCE-P1-1 | robots.txt 预检查 | `httpx.get(f"{scheme}://{host}/robots.txt")` + parser | Disallow 路径 → 400 |
| COMPLIANCE-P1-2 | Crawl-delay 遵守 | 解析 robots.txt Crawl-delay 值，设置请求间隔 | 遵守 10s delay |
| RATE-P1-1 | 每用户频率限制 | Django REST Framework throttle: 10/小时/用户 | 11 次请求 → 429 |
| RATE-P1-2 | User-Agent 设置 | `EY-Onboarding-AI-Crawler/1.0 (+https://ey.com/bot)` | robots.txt 可识别 |
| RATE-P1-3 | 重试退避 | exponential backoff: 1s → 2s → 4s | 失败后自动退避 |
| DEDUP-P1-1 | SimHash 去重 | `simhash` Python 库 + 3-bit 阈值 | 近重复文档 → 跳过 |
| COPYRIGHT-P1-1 | 来源 URL 记录 | CrawledDocument.source_url (URLField) | 元数据可溯源 |
| COPYRIGHT-P1-2 | "仅内部参考"标记 | CrawledDocument.internal_only (BooleanField) | Admin 可标记 |

### 7.3 P2 级要求（上线后一个月内实现）

| # | 要求 | 技术方案 | 验证方法 |
|---|---|---|---|
| RATE-P2-1 | 单域名频率限制 | per-domain 令牌桶（1请求/10s/域名） | 同域名密集爬取 → 429 |
| COPYRIGHT-P2-1 | 许可证检查提示 | Admin 爬取时显示版权提醒 | UI 提醒 |
| COPYRIGHT-P2-2 | 内容下架机制 | `Document.status = 'withdrawn'` 一键操作 | 投诉后可下架 |
| CONTENT-P2-1 | 编码校验 | UTF-8 校验 + 自动检测 | 非 UTF-8 内容 → 400 |

## P0：立即可做（1-2天）— 关键Bug与数据修复

### P0-1：i18n ZH 缺失键补全

- **问题**：v3.3-BUG-001 — ZH/common.json 缺少 `user_menu` 和 `error_title`
- **严重程度**：🟡中
- **修复方案**：
  1. 在 ZH/common.json 中添加 `user_menu: "用户菜单"` 和 `error_title: "错误"`
  2. 同时移除 `error_network` 重复键（v3.3-BUG-002）
  3. 使用 UTF-8 无 BOM 编码保存文件（v3.3-UX-002）
- **预期改善**：中文模式双语一致性恢复，消除英文回退显示
- **涉及文件**：`frontend/src/i18n/locales/zh/common.json`
- **验证方法**：切换中文模式，确认所有文案为中文无英文回退

### P0-2：侧边栏搜索胶囊样式修复

- **问题**：v3.3-UX-001 — UX-005 视觉回归：搜索框高度22.75px而非32px
- **严重程度**：🟡中
- **修复方案**：
  1. 在 Input 组件 style 中添加 `minHeight: 36px` 或 `padding: '8px 16px'` 补偿 `border: 'none'` 压缩
  2. 或将 borderRadius: 20 的胶囊样式改为 borderRadius: 12（减少空间压缩）
  3. 或改用 AntD `Input.Search` 组件自带样式
- **预期改善**：搜索框视觉大小恢复到"middle"级别，提升可发现性和可操作性
- **涉及文件**：`frontend/src/layout/AppLayout.tsx`（行355-371）
- **验证方法**：Playwright 测量搜索框高度 >= 30px

## P1：短期规划（1-2周）— 遗留UX打磨与交互补齐

### P1-1：Profile 空值字段友好展示

- **问题**：v3.3-UX-003 — 空值字段 `'—'` 不够友好
- **修复方案**：
  1. 将 `'—'` 改为 i18n 翻译键 `field_not_set`（EN: "Not set", ZH: "暂未设置"）
  2. 空值文字使用 `var(--color-text-tertiary)` 颜色 + italic 样式
  3. 可考虑添加 hover tooltip："可在设置中修改"（如果后续支持编辑）
- **涉及文件**：`frontend/src/pages/ProfilePage.tsx`

### P1-2：消息操作移动端可见性（V3.1 P2-2 遗留）

- **问题**：V3.1 P2-2 — hover 显示的 copy/share/regenerate 在移动端不可发现
- **修复方案**：
  1. 移动端：消息底部始终显示简洁操作行（copy icon + regenerate icon）
  2. 桌面端：保留 hover 行为
  3. 长按操作保留为补充方式
- **涉及文件**：`frontend/src/components/chat/MessageBubble.tsx`

### P1-3：History 页面增强（V3.1 P2-3 遗留）

- **问题**：V3.1 P2-3 — HistoryPage 搜索和筛选体验需优化
- **修复方案**：
  1. 添加全文本搜索（搜索消息内容而非仅标题）
  2. 优化时间筛选标签视觉
  3. 添加 "最近对话" 快捷入口
- **涉及文件**：`frontend/src/pages/HistoryPage.tsx`

### P1-4：知识库上传 UX 优化（V3.1 P2-4 遗留）

- **问题**：V3.1 P2-4 — 上传进度和状态反馈不够直观
- **修复方案**：
  1. 添加拖拽上传支持

## P3：长期规划（架构与体系）— 1-3月

### P3-1：状态管理优化（V3.1 P3-1 保留）

- 当前 chatStore 使用 Zustand，但 sessions 和 messages 之间存在竞态条件
- 建议：添加 optimistic update lock、debounce session switch、message batching

### P3-2：可访问性全面提升（V3.1 P3-2 保留）

- 当前已有基础，但未通过 WCAG 2.1 AA 级认证
- 建议：focus trap in modal、keyboard shortcuts for chat operations、high contrast mode、screen reader testing

### P3-3：组件重构与设计系统统一（V3.1 P3-3 保留）

- globals.css 超过 600 行
- 建议：提取为 CSS Modules 或 styled-components、集中 AntD token 覆盖、添加 Storybook

### P3-4：性能优化（V3.1 P3-4 保留）

- 建议：route-based lazy loading、SSE 连接复用、MessageBubble memo + 虚拟滚动

### P3-5：错误监控与日志（V3.1 P3-5 保留）

- 建议：Sentry error tracking、Web Vitals monitoring、API health check dashboard

## 风险评估

| 风险 | 等级 | 说明 |
|------|------|------|
| 搜索胶囊修复可能影响其他样式 | 🟡中 | 修改 Input padding/borderRadius 需确保不影响其他 Input 组件 |
| i18n BOM 移除可能导致编码问题 | 🟢低 | 只需用正确编码保存文件 |
| chatStore thinkingPhase 在慢网络下效果 | 🟡中 | 需真实浏览器在不同网络条件下验证 |
| 新增 data-testid 可能增加代码维护成本 | 🟢低 | 测试属性与业务逻辑分离，影响极小 |

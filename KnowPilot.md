## 应用业务场景

面向专业服务团队的知识与学习场景。团队成员需频繁查阅准则条款、方法论、行业规范、项目经验等专业知识，传统方式依赖手册翻阅、同事请教和分散文档搜索，效率低且答案不一致。KnowPilot 以 RAG（检索增强生成）为核心，将组织知识统一入库，把应用抽象为可对话、可检索、可追溯的知识库应用 Agent：AI 秒级精准回答并附带来源引用，同时提供知识库管理（文档上传 / 文档维护，当前版本不含 Web 爬虫）、管理后台（系统监控 / 权限管控）、合规审计四大能力。

**持续更新知识库，同一平台即可覆盖以下全部场景：**

| 场景                       | 知识库内容                                 | 典型问答                                                              |
| -------------------------- | ------------------------------------------ | --------------------------------------------------------------------- |
| **入职培训助手**     | HR 维护入职宝典                            | "Canvas 怎么配置？""报销发票怎么分类？""打印机怎么连？"               |
| **审计项目组 AI**    | 项目关键信息、会议纪要、客户背景、判断逻辑 | "这个调整为什么这么做？""客户去年拒函的原因是什么？"                  |
| **审计方法论查询**   | 审计方法论文档                             | "TOC/TOD/WT 分别怎么做？""抽凭的标准程序是什么？"                     |
| **准则问答助手**     | 港股/A 股准则原文                          | "IFRS 15 收入确认五步法怎么适用？""CAS 22 减值预期损失模型如何操作？" |
| **跨准则比较**       | CAS + HKFRS + IFRS 条款                    | "CAS 与 HKFRS 对租赁确认的差异是什么？"                               |
| **重大错报风险识别** | 历年风险案例 + 行业特征                    | "制造业客户典型风险领域有哪些？"                                      |
| **审计底稿编制**     | 标准底稿模板 + 优秀范例 + 填写指引         | "这份底稿的格式要求是什么？哪些字段必须填？"                          |
| **独立性合规检查**   | 独立性规则 + 违规案例                      | "团队成员持有客户股票怎么办？"                                        |
| **审计意见决策**     | CAS 1501 等意见判断标准                    | "什么情况下出保留意见？"                                              |

## 作品背景及内容

### ① 解决的实际业务问题

**知识跟着人走了，项目经验归零——新人对着底稿"考古"**

核心 Manager 离职，带走了某 IPO 项目的所有关键判断逻辑和客户沟通语境。新承接的同事对着几十个文件夹的底稿，不知道当初为什么做了这些调整，上下文完全断裂。项目经验本应是组织最宝贵的资产，却永远沉淀在离职员工的脑子里，一离职就归零。

**新人"社死"现场，入职第一周就想逃？**

安永审计新生小陈第一天进项目组，被安排抽凭。完全不知道从哪里下手。想问旁边的高级审计员，对方正在和客户电话 battle，根本无暇他顾；公司电脑连打印机怎么配置？IT 热线永远排队；加班打车报销，光发票分类和 description 就填错三次被打回；不知道该要什么资料？底稿不知道怎么编写？TOC/TOD/WT 该怎么做？熬到报告日，经理丢来一句"把需要 archive 的文件打印出来"，他根本不知道 archive checklist 在哪里、清 Q 该找谁签。

新人不是不努力，是组织的知识没有为他铺路。每一次"社死"都在消耗归属感，也在消耗团队本已紧张的审计预算。

**背后的共性根因：专业知识获取难、答案不一致、知识易过期、传承断裂**

准则条款、方法论、行业规范散落在手册、内网、PDF 中，翻阅耗时且查不全；同一问题不同人给出不同解读，专业判断标准缺乏统一性；制度更新后旧文档不及时同步，团队依过期信息做判断存在风险；项目经验和判断沉淀在个人脑中，人员流动后知识流失。

### ② 使用工具前后的变化

使用 KnowPilot 后，每个项目组都拥有了自己的专属 AI 知识库——审计一部有它的准则问答和底稿指引，审计二部有它的 IPO 案例库和历史判断记录，人才团队有它的入职宝典和培训体系，咨询团队有它的行业方法论和交付标准。同一个平台，千万个独立知识空间，各自沉淀、各自进化。

新人入职对着对话框问一句就能搞定 Canvas 配置和报销流程；项目经理对着自己的库追问项目历史决策依据；合伙人切换团队空间即刻获取不同业务线的关键洞察。每个团队都在自己的知识航道上，由 AI 精准领航，互不干扰又随时可共享——一套 KnowPilot，让组织的每一个作战单元都长出自己的大脑。

### ③ 推广价值 / 可复制性

KnowPilot 是一个**"一次部署、无限裂变、全场景复用"**的知识库应用 Agent 与知识领航引擎。验证一个团队的价值，就验证了全组织的价值；复制到一百个团队，不需要一百倍的投入。

**架构上，知识与业务解耦**——一套引擎服务万千团队，边际成本趋近于零。持续更新知识库即可从入职培训无缝切换至准则问答、风险识别、底稿编制等任意场景，边际部署成本趋近于零。

**推广上，以团队为颗粒度无限裂变**——价值验证后可无限复制，无需重复部署。一个审计组验证了 KnowPilot 的价值，全所 200 个审计组即可一键开通各自的知识空间，各自沉淀、各自进化。

**市场上，自带合规基因**——RBAC 细粒度权限、全操作审计日志、空间级数据隔离三重保障，天然适配金融、审计、医疗、政务等知识密集且合规门槛最高的行业——这些行业恰恰是专业问答需求最旺盛、付费意愿最强的市场。

**应用上，覆盖全业务线场景**——一份投入多线产出。入职培训、准则问答、项目组 AI、方法论查询、风险识别、底稿编制、独立性检查、意见决策……全场景一套平台，无需为每个场景单独采购或开发。

**决策上，ROI 清晰量化**——每条准则查询从数小时缩短至秒级，专业判断一致性从"人言人殊"升级为"全员统一标准答案"，项目经验从"个人资产"转化为"组织资产"，知识维护从人力驱动变为自动索引。极大降低推广阻力，让决策者一眼看懂价值。

## Tech Stack

- **Backend**: Django 5.0 + DRF + Celery + Redis
- **Frontend**: React 18 + TypeScript + Vite + Ant Design 5 + Zustand
- **LLM**: Qwen via DashScope API (OpenAI 兼容协议)
- **Embeddings**: Qwen text-embedding-v4 (1024-dim)
- **Vector DB**: pgvector (PostgreSQL 16)
- **RAG**: LangChain + Docling
- **Infra**: Docker Compose 一键部署
